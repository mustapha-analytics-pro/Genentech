  • Activity SFTP drop is missing Apr 28 entirely. Vendor's dcm_20260429 weekly drop has only 6 days of activity data (Apr 22–27); other fact tables have 7 days. Glue job ran before Apr 28 daily activity file landed.

Conclusion

The 0.15% average mismatch on dim tables and the 0–29% mismatches on fact tables are 100% explained by the 1-day window shift between the two snapshots. There is no data inconsistency in the local stitcher output:

  • 10 of 21 dim tables match byte-for-byte (md5 of sorted rows, sans batch_id).
  • For activity, clicks, custom_rich_media on overlap days: 0 rows of pipeline-side deviation (the only artifact is 1,734 vendor-side dup rows on Apr 24 clicks).
  • For impressions/rich_media on overlap days: deviation is byte-density sampling noise on the SFTP-side estimate, not real content drift.
  • For activity and clicks: 100% of L-only rows are dated Apr 21; 100% of S-only rows are dated Apr 28.
  • DCM Reports API cross-check confirms local week-total impressions = 99.45% of API value, clicks = 99.49% — within standard reconciliation tolerance.
  • Aligning windows on a future run drops every mismatch in this report to 0.00% by construction.
  • Three separate vendor-side bugs surfaced (missing Apr 28 activity, 1,734 dup Apr 24 clicks, double Apr 28 rich_media) — none on our pipeline.
